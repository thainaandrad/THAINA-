# THAINA-
1. Organização de Pastas
Para que o projeto fique limpo, sugiro a seguinte hierarquia:

/src: Código-fonte dividido por categoria (ex: sorting/, graphs/).

/docs: Notas teóricas sobre a complexidade de tempo e espaço.

README.md: Sua "planilha" principal de progresso.

2. O README (Sua Planilha de Acompanhamento)
Abaixo está o código Markdown para a sua tabela de progresso. Você pode atualizar o status com emojis (✅, 🏗️, ❌) conforme avança.

3. Como usar este projeto no GitHub
Crie um novo repositório: Nomeie como study-algorithms.

Adicione o README.md: Use o modelo acima.

Use Issues para Checklist: Crie uma "Issue" para cada algoritmo que deseja aprender. Isso gera um histórico visual de tarefas.

Cards de Projetos (Kanban): Vá na aba Projects do repositório e crie um quadro com as colunas: To Do, In Progress, Done. Isso substitui qualquer planilha de gestão de tempo.
